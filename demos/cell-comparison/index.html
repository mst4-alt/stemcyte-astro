Cell Animation — Tech Comparison

Same visual, different rendering technology. Pick the one that feels right.

CSS / SVG

CSS/SVG

Variant A: Animated SVG Circles

Pure CSS + SVG, zero JavaScript. SVG circles with radial gradients animated via @keyframes. Layered blur filters fake depth. Simplest, lightest approach — no collision or mouse interaction.

View Demo

CSS/SVG

Variant B: CSS Houdini Custom Properties

SVG circles driven by @property-registered custom properties with independent timing per axis. More organic, non-repeating motion than Variant A. Still zero JS.

View Demo
Canvas 2D

Canvas 2D

Variant A: Direct Port

Clean recreation of the science hero. 3-layer depth, drawCell with nucleus/specular/glow, brownian drift, soft collision physics, breathing animation. Plus gentle mouse repulsion.

View Demo

Canvas 2D

Variant B: Perlin Noise Flow

Same cell rendering, but driven by a Perlin noise flow field instead of brownian motion. Coordinated, river-like movement. Cursor creates a gentle vortex disturbance.

View Demo
WebGL (Custom Shaders)

WebGL

Variant A: Metaball Shader

Raw WebGL with GLSL fragment shader. SDF metaballs that merge and separate organically. Mouse attracts/repels blobs. Zero dependencies.

View Demo

WebGL

Variant B: Volumetric Raymarched Cells

Raymarched sphere SDFs with inner nucleus, fake subsurface scattering at edges. Translucent, biological look that Canvas 2D can't achieve. Zero dependencies.

View Demo
Pixi.js

Pixi.js

Variant A: Sprite-Based Cells

GPU-accelerated 2D sprites with programmatic gradient textures. Pixi blur filters per depth layer. 30-40 cells across 3 layers with brownian motion + collision. Mouse repulsion.

View Demo

Pixi.js

Variant B: Mesh + Custom Shader

Procedural cell rendering via Pixi mesh shaders — resolution-independent, sharp at any size. Displacement filter for "looking through water" effect. Mouse interaction via uniform.

View Demo
Three.js

Three.js

Variant A: Instanced Spheres

InstancedMesh with custom ShaderMaterial — Fresnel edge glow, nucleus, radial gradient body. OrthographicCamera for 2D microscope feel. Subtle mouse tilt.

View Demo

Three.js

Variant B: Particle System

Points particle system with custom point-sprite ShaderMaterial. 50+ cells across 4 layers. Lightweight, high cell count. Mouse repulsion as a force on nearby particles.

View Demo
GSAP

GSAP

Variant A: DOM Elements

Cells as real DOM divs with CSS radial-gradient and box-shadow. GSAP animates position/scale with long durations and yoyo. ~15 cells, no canvas. Mouse via gsap.quickTo.

View Demo

GSAP

Variant B: Canvas Orchestration

Canvas 2D drawCell rendering, but all motion orchestrated by GSAP timelines with MotionPathPlugin for curved paths. Buttery easing that's hard to replicate with manual physics.
